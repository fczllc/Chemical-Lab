# 元素探索者 (Element Explorer Kids) 安装手册

本文档面向需要在不同环境中部署「元素探索者」应用的运维人员或开发者，提供三种部署方式的详细步骤：本地命令行部署、Docker 容器化部署、Cloudflare Pages 云端部署。

---

## 环境要求

所有部署方式均需要以下前置条件：

- **Node.js**：v18 或更高版本（推荐 v20 LTS）
- **npm**：v9 或更高版本（随 Node.js 一同安装）
- **Git**：用于克隆代码仓库

验证环境：

```bash
node -v    # 应输出 v18.x.x 或更高
npm -v     # 应输出 9.x.x 或更高
git --version
```

---

## 一、本地命令行部署（Linux 一键安装）

适用于：快速在 Linux 服务器或本地 Linux 环境启动应用。

### 1.1 获取代码

```bash
git clone https://github.com/fczllc/Chemical-Lab.git
cd Chemical-Lab
```

### 1.2 一键安装

项目根目录下执行：

```bash
chmod +x scripts/install-linux.sh
./scripts/install-linux.sh
```

脚本会自动完成以下操作：
1. 检查 Node.js 版本（要求 ≥ v18）
2. 安装项目依赖（优先使用 `npm ci`，无 lock 文件时回退到 `npm install`）
3. 执行生产构建并运行烟雾测试，验证部署环境

### 1.3 启动服务

安装成功后，使用以下命令启动：

```bash
# 开发模式（带热更新）
npm run dev

# 生产预览模式
npm run preview
```

默认访问地址：
- 开发模式：`http://localhost:5173/`
- 生产预览：`http://localhost:4173/`

### 1.4 常用命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动 Vite 开发服务器 |
| `npm run build` | 构建生产版本 |
| `npm run preview` | 预览生产构建 |
| `npm run deploy:local` | 构建并启动生产预览 |
| `npm run deploy:local -- --smoke-test` | 构建并验证，不保留服务 |

## 二、Docker 容器化部署

适用于：需要隔离环境、标准化部署或集成到容器编排平台的场景。

### 2.1 前置要求

- 安装 [Docker](https://docs.docker.com/get-docker/)（v20.10+）
- 安装 [Docker Compose](https://docs.docker.com/compose/install/)（v2.0+，通常随 Docker Desktop 一同安装）

### 2.2 快速启动

在项目根目录执行：

```bash
docker compose up --build -d
```

首次构建需要下载基础镜像并编译应用，耗时约 1-3 分钟（取决于网络）。

### 2.3 访问应用

构建完成后，打开浏览器访问：

```
http://localhost:8080/
```

### 2.4 常用命令

| 命令 | 说明 |
|------|------|
| `docker compose up --build -d` | 构建并后台启动 |
| `docker compose down` | 停止并移除容器 |
| `docker compose logs -f` | 查看实时日志 |
| `docker compose ps` | 查看容器状态 |

### 2.5 自定义端口

如需修改映射端口，编辑 `docker-compose.yml`：

```yaml
services:
  web:
    build: .
    ports:
      - "3000:80"   # 将宿主机的 3000 端口映射到容器的 80 端口
    restart: always
```

修改后执行 `docker compose up -d` 重新加载。

## 三、Cloudflare Pages 云端部署

适用于：需要全球 CDN 加速、无需维护服务器的场景。

### 3.1 前置要求

- [Cloudflare](https://dash.cloudflare.com/) 账号
- 安装 [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/install-and-update/)：

```bash
npm install -g wrangler
```

- 登录 Cloudflare：

```bash
npx wrangler login
```

### 3.2 通过 Cloudflare 网页端部署（推荐新手）

适用于：不熟悉命令行，或希望使用图形界面快速部署的用户。

#### 步骤 1：登录 Cloudflare Dashboard

1. 访问 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 使用你的 Cloudflare 账号登录

#### 步骤 2：创建 Pages 项目

1. 在左侧导航栏点击 **Pages**
2. 点击 **创建项目** 或 **Create a project**
3. 选择 **连接到 Git**（Connect to Git）
4. 授权 Cloudflare 访问你的 GitHub 账号
5. 选择你的仓库（例如 `Chemical-Lab`）

#### 步骤 3：配置构建设置

在构建设置页面填写以下信息：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 项目名称 | `chemical-lab` | 自定义，将部署到 `xxx.pages.dev` |
| 生产分支 | `master` | 你的默认分支名称 |
| 框架预设 | `无` | 选择 "None" |
| 构建命令 | `npm run build` | Vite 构建命令 |
| 构建输出目录 | `dist` | Vite 默认输出目录 |

> **注意**：构建输出目录填写 `dist` 即可，不要加斜杠。Cloudflare 会自动在前面加上 `/`。

#### 步骤 4：保存并部署

1. 点击 **保存并部署**（Save and Deploy）
2. Cloudflare 会自动开始首次构建和部署
3. 等待构建完成（通常 1-2 分钟）

#### 步骤 5：访问你的站点

部署成功后，Cloudflare 会提供一个默认域名：

```
https://chemical-lab.pages.dev
```

你也可以在 **自定义域** 中添加自己的域名。

#### 常见问题

**Q: 构建失败，提示 "workers-site/index.js" 未找到？**

A: 这是因为你选择了 Workers/Pages Functions 模式。解决方式：
- 检查构建设置中的 **构建输出目录** 是否为 `dist`
- 确保没有启用 Pages Functions（Functions 设置应为空）
- 如果使用了 `wrangler.toml`，确保配置正确：
  ```toml
  [site]
  bucket = "dist"
  ```

**Q: 构建成功但页面显示 404？**

A: 这是单页应用（SPA）的路由问题。在 `dist/` 目录中添加 `_routes.json`：

```json
{
  "version": 1,
  "include": ["/*"],
  "exclude": ["/assets/*"]
}
```

或在项目根目录的 `public/` 文件夹中创建该文件，Vite 构建时会自动复制到 `dist/`。

**Q: 构建警告 "Some chunks are larger than 500 kB"？**

A: 这是警告不是错误，不会导致构建失败。原因是 Three.js 等库体积较大。如需优化，可在 `vite.config.js` 中添加代码分割配置。

### 3.3 通过命令行部署

适用于：熟悉命令行，或需要集成到 CI/CD 流程的用户。

在项目根目录执行：

```bash
node scripts/deploy-cloudflare.mjs
```

脚本会自动完成：
1. 执行 `npm run build` 构建生产版本
2. 检查 wrangler CLI 是否已安装
3. 使用 `wrangler pages deploy dist/` 部署到 Cloudflare Pages

### 3.4 烟雾测试

如需验证构建但不实际部署：

```bash
node scripts/deploy-cloudflare.mjs --smoke-test
```

### 3.5 自动部署（GitHub Actions）

项目已包含 `.github/workflows/deploy-cloudflare.yml`，配置完成后可实现推送即部署：

1. 在 GitHub 仓库设置中添加以下 Secrets：
   - `CLOUDFLARE_API_TOKEN`：Cloudflare API 令牌（需包含 "Cloudflare Pages:Edit" 权限）
   - `CLOUDFLARE_ACCOUNT_ID`：Cloudflare 账户 ID

2. 获取 API 令牌：
   - 访问 [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)
   - 创建令牌，模板选择 "Edit Cloudflare Workers"
   - 权限：Account > Cloudflare Pages > Edit
   - 账户资源：包含目标账户

3. 获取账户 ID：
   - 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
   - 右侧边栏即可看到 **Account ID**

4. 推送代码到 `main` 分支即可触发自动部署。

### 3.6 Wrangler 配置

`wrangler.toml` 已包含基础配置：

```toml
name = "element-explorer-kids"
compatibility_date = "2026-06-02"

[site]
bucket = "dist"

[build]
command = "npm run build"
```

## 五、常见问题

### Q1: 安装脚本提示 Node.js 版本过低？

请升级 Node.js 至 v18 或更高版本。推荐使用 [nvm](https://github.com/nvm-sh/nvm) 管理版本：

```bash
nvm install 20
nvm use 20
```

### Q2: `npm ci` 失败？

如果 `package-lock.json` 与 `package.json` 不一致，尝试删除 lock 文件后重新安装：

```bash
rm package-lock.json
npm install
```

### Q3: Docker 构建卡住或超时？

可能是网络问题导致基础镜像下载缓慢。尝试更换 Docker 镜像源或使用代理：

```bash
# 配置 Docker 镜像加速（编辑 ~/.docker/daemon.json）
{
  "registry-mirrors": ["https://mirror.ccs.tencentyun.com"]
}
```

### Q4: Cloudflare 部署提示未授权？

确认已执行 `npx wrangler login` 完成登录，且 API 令牌具有足够的权限。

### Q5: 如何修改应用端口？

- **本地部署**：Vite 默认使用 5173（开发）和 4173（预览），可通过 `--port` 参数修改
- **Docker 部署**：修改 `docker-compose.yml` 中的端口映射
- **Cloudflare Pages**：无需关心端口，由平台自动分配
