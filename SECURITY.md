# 安全政策

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 0.1.x | ✅ 支持 |
| < 0.1.0 | ❌ 不再支持 |

## 报告漏洞

如果你发现了安全漏洞，请通过以下方式报告：

1. **不要** 在公开的 GitHub Issues 中报告安全漏洞
2. 发送邮件至项目维护者（通过 GitHub 个人资料获取联系方式）
3. 提供详细的漏洞描述和复现步骤

## 响应时间

- 确认收到报告：48 小时内
- 初步评估：7 天内
- 修复和发布：根据严重程度，通常在 30 天内

## 安全最佳实践

使用本项目时，请注意：

- 不要在客户端暴露敏感 API 密钥
- 定期更新依赖项
- 使用 HTTPS 部署生产环境
- 遵循最小权限原则

## 致谢

感谢所有负责任地披露安全问题的贡献者。
